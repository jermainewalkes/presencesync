PresenceSync — Setup Guide

Keep your Microsoft Teams presence and your Slack status in sync, both ways, on macOS or Windows. Setup is a one-time job — register an app with Microsoft and one with Slack, then connect them here. About 15 minutes. If your organisation already has the apps registered, skip to step 3.

Before you start: both steps need consent from an administrator of your Microsoft 365 tenant and your Slack workspace. If you're not an admin, you'll need one to approve the apps.

1 · Microsoft (Teams)

In the Azure portal → App registrations:

  1. Open your PresenceSync app registration (or create a new one).
  2. Authentication → Add a platform → Mobile & desktop applications → add the redirect URI http://localhost, and turn on "Allow public client flows".
  3. API permissions → Add a permission → Microsoft Graph → Delegated → add Presence.Read and Presence.ReadWrite.
  4. Click Grant admin consent for your tenant.
  5. From Overview, copy the Application (client) ID and the Directory (tenant) ID — you'll paste these into PresenceSync.

2 · Slack

At api.slack.com/apps:

  1. Create New App → From scratch → name it PresenceSync → pick your workspace.
  2. OAuth & Permissions → Redirect URLs → add exactly http://localhost:53682/callback → Save URLs.
  3. Same page → User Token Scopes → add users.profile:read, users.profile:write, and users:read.
  4. Install to Workspace and approve (an admin may need to approve the app).
  5. Basic Information → App Credentials → copy the Client ID and Client Secret.
Token rotation is optional — PresenceSync auto-refreshes your Slack token either way, so you never have to fetch a key again after this.

3 · Connect in the app

  1. Launch PresenceSync — on first run the Settings window opens automatically (or click the menu-bar/tray icon → Settings…).
  2. Enter your Microsoft Tenant ID and Client ID, and your Slack Client ID and Client Secret, then click Save. (If you were given an org-config.json, drop it in the project folder before first run and these fields fill themselves.)
  3. Click Connect Microsoft, then Connect Slack — each opens a browser window to sign in. Your tokens are stored in the OS keychain (macOS Keychain / Windows Credential Locker).
  4. Click Test connection — you want both lines to report OK.
  5. Turn on Start at login from the menu so it's always running.

The menu-bar/tray icon shows health at a glance: the plain glyph when all's well, an amber dot if something's expiring, a red dot if it needs attention.

For organisation admins

Register both apps once — every user in your Microsoft tenant and Slack workspace can then use the same four IDs (each person still signs in as themselves). Copy org-config.example.json to org-config.json, fill in the values, and share it internally only — it contains your Slack client secret, so never publish or commit it.

Troubleshooting

  1. Slack sign-in fails instantly — the redirect URL must be exactly http://localhost:53682/callback, and the Client ID + Secret must be saved first.
  2. Teams won't go Busy from a Slack huddle — check that Presence.ReadWrite has admin consent, and that you're signed into a Teams client.
  3. Red dot / "reconnect needed" — open Settings and click Connect again.
  4. Keychain prompts — click Always Allow; that's where your tokens live.
